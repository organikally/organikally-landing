# Accessibility (WCAG 2.1 AA)

Owner: `accessibility-specialist`. Status: **Phase 5 pass — automated AA green; manual checks done.**

## Automated

- **Lighthouse Accessibility = 100** (mobile) on home and journal post.
- Earlier failures fixed: low-opacity cream footer text bumped to `/70`–`/75` (was `/50`–`/60`)
  so all text meets AA on the forest footer.

## Contrast over the moving video (the hard case)

Hero/interstitial copy sits over a scrubbing canvas whose frames change. Legibility is held by:
- A vertical forest scrim (`rgba(14,59,20, .55→.80)`) **plus** a left/bottom-anchored scrim
  (`from-forest/90`) so the copy column is effectively over near-solid forest regardless of frame.
- Cream/white text + a `text-shadow` for margin.
- Verified visually at multiple scroll positions and on mobile: the lightest source frame
  (frame 1, light background) still renders cream copy with strong contrast. Hero headings are
  large text (≥24px) → AA threshold 3:1, comfortably exceeded by the forest scrim.

## Keyboard & focus

- **Skip-to-content** link (first focusable; visible on focus) → `#main` on every page.
- Global `:focus-visible` ring (2px gold-ink, 2px offset).
- FAQ accordion: real `<button>` with `aria-expanded` + `aria-controls`; content stays in the DOM.
- Mobile menu: `aria-label`, `aria-expanded`, `Esc` to close, focusable controls.
- All interactive elements are native `<a>`/`<button>`.

## Semantics & media

- Landmarks: `<header>`, `<main id="main">`, `<nav aria-label>`, `<footer>`.
- One `h1` per page; logical `h2/h3` order.
- Decorative hero canvas/poster wrapper is `aria-hidden`; product/logo images have descriptive alt.
- Icon-only controls have `aria-label`; decorative icons are `aria-hidden`.

## Motion

- `prefers-reduced-motion: reduce`: the hero **disables the scrub** and shows the static poster;
  `Reveal` renders content immediately (no transform); a global CSS rule zeroes transitions/animations.
  (Code-verified; confirm on a real device in final QA.)
